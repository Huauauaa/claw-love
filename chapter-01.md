# 第一章：Bug 与初遇

凌晨一点四十七分，林知夏盯着屏幕，眼睛被 IDE 的深色主题烤得发干。

“接口又改了？”她把键盘敲得哒哒响，在公司协作群里发出第三条消息：  
「/openclaw/grab/status 字段为什么从 `taskId` 变成 `job_id`，没人通知前端？」

没人回。  
只回了一个系统提醒：周叙白已将群消息设为免打扰。

林知夏，00 后，刚转正三个月，前端组新晋“修图救火双料选手”。她不怕改需求，不怕赶上线，就怕“后端悄悄改字段还装没事”。

她合上咖啡杯盖，站起来，抱着电脑冲向后端工位区。

周叙白正坐在最里面，白衬衫袖子挽到手肘，屏幕上开着三排日志。旁边同事小声说：“别惹周叙白，他今晚在追线上丢包。”

林知夏偏不信邪，直接把电脑转到他面前：“你改了接口，前端页面全挂了。”

周叙白抬眼，声音冷静得像机器播报：“不是改，是规范。`taskId` 不符合命名约定。”

“你规范之前能不能先通知人？”

“文档更新了。”

“更新在你个人分支 README 里，谁看得到？”

两人对视三秒，空气里像有静电噼啪响。  
旁边同事默默戴上耳机，假装自己是盆栽。

第二天上午，公司全员会上，CTO 宣布：  
“我们要参加今年开源技术挑战赛，项目代号 Open Claw。目标很简单——让机械爪抓取系统开放给开发者，三周出可演示版本。”

会议室一阵骚动，林知夏听见有人小声吐槽：“三周？你不如让我用 Python 写情书。”

项目名单投在大屏上：  
前端负责人——林知夏。  
后端负责人——周叙白。

林知夏看向周叙白，周叙白也刚好抬头。  
两人同时别开视线。

Open Claw 启动第一天，冲突就没停过。

林知夏要做可视化控制台，强调“状态反馈必须实时”；  
周叙白要做任务队列容错，坚持“先保稳定，再谈体验”。

“你这按钮点下去三秒没反馈，用户会以为卡死。”林知夏皱眉。  
“比起动画，系统不崩更重要。”周叙白不抬头。  
“你们后端是不是觉得用户会读日志？”  
“你们前端是不是觉得世界靠 CSS 运行？”

群里沉默了十秒。  
然后产品经理发了个哭脸：「两位都对，先别离婚。」

真正的危机发生在演示前一晚。

测试环境里，Open Claw 死活抓不住目标物。机械臂每次即将闭合，都会偏移两厘米，像故意躲开。

会议室灯全亮着，窗外写字楼只剩零星几盏灯。  
林知夏盯着监控画面，连续回放了二十遍：“前端指令没问题，控制面板参数也没抖。”

周叙白盯日志：“任务调度时钟漂移，和视觉模块时间戳对不上。”

“那就对齐。”林知夏说。  
“要改三处服务。”周叙白说。  
“我陪你改。”

周叙白终于抬头看她，像是在确认这句话是不是客套。  
林知夏已经把外卖软件关了，开了新终端：“别看了，快点，天亮前要跑通。”

凌晨四点零二分，办公室安静得只剩风扇声和键盘声。

林知夏把最后一个重试逻辑补上，周叙白推送了新的时钟同步补丁。  
两人一起按下回车，监控画面里，机械爪缓慢下探——  
夹紧——  
抬升——  
目标物稳稳被举起。

林知夏先是愣了两秒，接着猛地拍桌：“成了！”

她转过头，想和周叙白击掌，却发现周叙白只是看着屏幕，嘴角很轻地扬了下。  
那种笑很短，像日志里一行一闪而过的成功信息。

“你干嘛不激动？”林知夏问。  
“激动。”周叙白停顿一下，声音低了点，“只是没想到你能把前端状态机写这么稳。”

林知夏扬眉：“你这算夸人吗？”  
周叙白把电脑合上一半，终于看向她：“算。你比我想的靠谱。”

窗外天色发白，第一班地铁快开了。  
林知夏忽然觉得，今晚最难调通的，可能不止是 Open Claw 的时钟。
