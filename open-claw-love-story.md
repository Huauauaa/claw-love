# 《抓到你了》故事设定

## 角色命名

- 女主 A：林知夏
- 男主 B：周叙白

## 四章节大纲

### 第一章：Bug 与初遇（起）

- A 是刚转正的 00 后前端工程师，B 是同公司后端骨干。
- 公司启动开源项目 Open Claw，A 与 B 被安排搭档。
- 两人因接口规范和节奏冲突不断，互相看不顺眼。
- 演示前夜系统抓取失败，他们被迫通宵联调。
- 最终系统跑通，关系从对立转向初步认可。

### 第二章：并肩开发（承）

- 高压迭代中，A 与 B 从互怼走向互补。
- Open Claw 在社区走红，二人共同处理 Issue 与文档。
- 线下分享会突发设备故障，B 临场接住 A 的内容。
- A 第一次明确心动，暧昧开始发酵。

### 第三章：误会与分岔（转）

- 公司推动 Open Claw 商业化，开源与闭源路线产生冲突。
- A 误以为 B 背离开源立场，两人爆发争执并短暂决裂。
- 发布日出现安全漏洞，舆论压力袭来。
- B 顶住内部压力提出折中方案，修复信任。

### 第四章：版本与余生（合）

- 新版本发布后口碑回暖，A 与 B 再次并肩。
- 两人在工作外建立更深连接，感情落地。
- 周年发布后，B 以 “长期协作” 的程序员式告白示爱。
- A 接受告白，故事收束于“并肩调试人生”。

## 第一章正文：Bug 与初遇

凌晨一点四十七分，林知夏盯着屏幕，眼睛被 IDE 的深色主题烤得发干。

“接口又改了？”她把键盘敲得哒哒响，在公司协作群里发出第三条消息：  
「/openclaw/grab/status 字段为什么从 `taskId` 变成 `job_id`，没人通知前端？」

没人回。  
只回了一个系统提醒：周叙白已将群消息设为免打扰。

林知夏，00 后，刚转正三个月，前端组新晋“修图救火双料选手”。她不怕改需求，不怕赶上线，就怕“后端悄悄改字段还装没事”。

她合上咖啡杯盖，站起来，抱着电脑冲向后端工位区。

周叙白正坐在最里面，白衬衫袖子挽到手肘，屏幕上开着三排日志。旁边同事小声说：“别惹周叙白，他今晚在追线上丢包。”

林知夏偏不信邪，直接把电脑转到他面前：“你改了接口，前端页面全挂了。”

周叙白抬眼，声音冷静得像机器播报：“不是改，是规范。`taskId` 不符合命名约定。”

“你规范之前能不能先通知人？”

“文档更新了。”

“更新在你个人分支 README 里，谁看得到？”

两人对视三秒，空气里像有静电噼啪响。  
旁边同事默默戴上耳机，假装自己是盆栽。

第二天上午，公司全员会上，CTO 宣布：  
“我们要参加今年开源技术挑战赛，项目代号 Open Claw。目标很简单——让机械爪抓取系统开放给开发者，三周出可演示版本。”

会议室一阵骚动，林知夏听见有人小声吐槽：“三周？你不如让我用 Python 写情书。”

项目名单投在大屏上：  
前端负责人——林知夏。  
后端负责人——周叙白。

林知夏看向周叙白，周叙白也刚好抬头。  
两人同时别开视线。

Open Claw 启动第一天，冲突就没停过。

林知夏要做可视化控制台，强调“状态反馈必须实时”；  
周叙白要做任务队列容错，坚持“先保稳定，再谈体验”。

“你这按钮点下去三秒没反馈，用户会以为卡死。”林知夏皱眉。  
“比起动画，系统不崩更重要。”周叙白不抬头。  
“你们后端是不是觉得用户会读日志？”  
“你们前端是不是觉得世界靠 CSS 运行？”

群里沉默了十秒。  
然后产品经理发了个哭脸：「两位都对，先别离婚。」

真正的危机发生在演示前一晚。

测试环境里，Open Claw 死活抓不住目标物。机械臂每次即将闭合，都会偏移两厘米，像故意躲开。

会议室灯全亮着，窗外写字楼只剩零星几盏灯。  
林知夏盯着监控画面，连续回放了二十遍：“前端指令没问题，控制面板参数也没抖。”

周叙白盯日志：“任务调度时钟漂移，和视觉模块时间戳对不上。”

“那就对齐。”林知夏说。  
“要改三处服务。”周叙白说。  
“我陪你改。”

周叙白终于抬头看她，像是在确认这句话是不是客套。  
林知夏已经把外卖软件关了，开了新终端：“别看了，快点，天亮前要跑通。”

凌晨四点零二分，办公室安静得只剩风扇声和键盘声。

林知夏把最后一个重试逻辑补上，周叙白推送了新的时钟同步补丁。  
两人一起按下回车，监控画面里，机械爪缓慢下探——  
夹紧——  
抬升——  
目标物稳稳被举起。

林知夏先是愣了两秒，接着猛地拍桌：“成了！”

她转过头，想和周叙白击掌，却发现周叙白只是看着屏幕，嘴角很轻地扬了下。  
那种笑很短，像日志里一行一闪而过的成功信息。

“你干嘛不激动？”林知夏问。  
“激动。”周叙白停顿一下，声音低了点，“只是没想到你能把前端状态机写这么稳。”

林知夏扬眉：“你这算夸人吗？”  
周叙白把电脑合上一半，终于看向她：“算。你比我想的靠谱。”

窗外天色发白，第一班地铁快开了。  
林知夏忽然觉得，今晚最难调通的，可能不止是 Open Claw 的时钟。

## 第二章正文：并肩开发

Open Claw 第一次对外公测那天，林知夏的手机像被塞进了爆米花机。

社区通知一条接一条往外蹦：  
「机械臂抓取轨迹在低分辨率下抖动」  
「文档里环境变量拼写不一致」  
「Windows 下 demo 跑不起来」

她刚把第一条 issue 标成 `confirmed`，第二十条已经刷出来了。

“火了。”产品经理声音发飘，“我们真的火了。”

“先别兴奋，先分锅。”林知夏把椅子一转，冲周叙白扬下巴，“后端十条，前端十条，文档一起背。”

周叙白看了她一眼，没抬杠，只把工单面板按优先级重新排好：“高危我先处理。你把复现步骤统一模板，不然社区会互相抄错答案。”

“收到，周老师。”

“别阴阳。”

“这是尊称。”

两人嘴上照旧互呛，手上却配合得越来越像一套完整系统。  
林知夏负责把外部反馈翻译成能执行的任务，周叙白负责把任务拆成可落地的服务修改。一个拉近用户，一个兜底系统。

三天后，Open Claw 的 Star 数破万。公司临时决定：周末去参加城市开发者沙龙，做一场线下分享。

“你讲前端交互和开发者体验，我讲架构和稳定性。”周叙白看着议程表说。

“行。”林知夏点头，“别临场给我加‘顺便讲一下控制算法原理’这种地狱任务。”

“我没那么坏。”

“你最好是。”

---

周六下午，活动场地人满得像春运候车厅。  
林知夏在后台插上电脑，投影黑屏。

她换了 HDMI 线，还是黑。  
换接口，还是黑。  
主持人在台上已经报幕：“下面有请 Open Claw 团队——”

林知夏额头冒汗，低声骂了句：“完了，这比线上宕机还社死。”

周叙白从她手里接过遥控笔，声音很稳：“你把备用机开起来，我先上。”

“可我的部分——”

“我看过你全部稿子。”他看着她，“你写的每一页我都看过。先把场子接住。”

他转身上台，灯光落下来，周叙白拿着话筒，第一句就不是自己的开场白：

“大家好，我是 Open Claw 后端负责人周叙白。今天这场分享，本来最精彩的部分来自我的搭档林知夏。设备有点小意外，我先替她热个场，等她上来你们再听完整版。”

台下笑了，气氛一下松开。

林知夏抱着备用电脑站在侧台，听他把她写的每一个要点讲得一字不差——  
为什么把状态分成 `idle/running/retrying`，  
为什么要在 UI 层显式暴露错误级别，  
为什么“开发者体验”不是花哨皮肤，而是降低踩坑成本。

那些她熬夜改过十几版的句子，从他嘴里说出来，居然比她自己还顺。

十分钟后，投影恢复。林知夏接过话筒，先看了周叙白一眼，才笑着说：

“大家好，我是林知夏。刚才那段不算彩排事故，算我们团队做了个实时容灾演示。”

台下爆发一阵掌声。

分享结束后，一群开发者围上来提问，林知夏连喝口水的时间都没有。等最后一位合影结束，场馆已经开始收椅子。

她和周叙白并肩走出会场，夜风有点凉。

“你什么时候背下我稿子的？”她问。

“你改第三版的时候。”周叙白把外套递给她，“你在会议室自言自语练过三遍，声音太大，我不想听都不行。”

“偷听还这么理直气壮？”

“不是偷听，是风险预案。”

林知夏笑出声：“你这个人，连安慰都要写成事故复盘。”

周叙白没接话，只是把步子放慢了半拍，配合她的节奏。

---

回程地铁上，人不算多。林知夏抱着电脑包坐在靠窗位置，刚说完一句“我今天真是差点心梗”，下一秒就困得睁不开眼。

她头一点一点，最后轻轻靠在周叙白肩上。

周叙白僵了半秒，没动。  
过了会儿，他把手机亮度调到最低，单手回社区评论：

「Issue #287 已复现，修复将于明日合并。感谢反馈。」  
「文档 typo 已更正，欢迎继续提 PR。」

车窗映出两个人的影子，一个睡着，一个清醒；一个呼吸均匀，一个连翻页都尽量放轻。

林知夏在半梦半醒间听见地铁报站，没睁眼，只含糊问了一句：“到哪了……”

周叙白低声回她：“还没到。你睡吧。”

那一刻她忽然很确定，自己对这个“人形代码规范”的情绪，已经不只是“合作愉快”那么简单。
